LOCATION SUBMIT FLOW ADDED

Actual workflow used:
1. Doctor scans QR code using mobile camera.
2. QR opens patient referral web form /l/<token>.
3. Doctor enters patient details.
4. On Submit button click, browser/app asks location permission.
5. Current latitude/longitude is captured only once at submit time.
6. Lead/referral saves patient + doctor + hospital + date/time + location + status.
7. Doctor mobile app patient detail page shows location status and Google Maps link.

DB columns added to leads table:
- submit_latitude
- submit_longitude
- submit_accuracy
- submit_location_status
- location_captured_at

Important:
- Browser geolocation needs HTTPS/domain for normal mobile Chrome.
- No background location is used.
