Live QR + PWA Fix Notes
=======================

Updated in this ZIP:

1) Live Render QR URL
- config.py default BASE_URL changed to https://kimssaveera.onrender.com
- .env BASE_URL changed to https://kimssaveera.onrender.com
- clinic QR image is regenerated from current BASE_URL when opening /clinic/created/<id>
- This corrects old QR images/links that were generated with 127.0.0.1.

2) Create QR doctor dropdown
- /clinic/api/pending-doctors/<hospital_id> now returns only approved, active doctors who do not already have a generated QR.
- A doctor with a generated QR is excluded globally from future QR dropdowns.
- Duplicate checks block QR creation by doctor id, mobile, or PAN.

3) Doctor Approval QR status sync
- Doctor Approval page now calculates QR status from doctor_hospital_qr table using actual_qr_generated.
- users.qr_generated is also updated when QR is created for backward compatibility.

4) Referral form submit + location
- Patient referral form now has a visible location status box.
- On submit, browser requests approximate geolocation.
- submit_latitude, submit_longitude, submit_accuracy, submit_location_status, and location_captured_at are posted and saved.
- If location is denied/unavailable, form still submits with status denied_or_unavailable.

5) Submit success close screen
- Submitted page removes home navigation from successful submission.
- It attempts to close the page automatically and blocks browser back navigation on the submitted page.
- Browser security may prevent auto-close if the page was not opened by script; user then sees a Close Page button.

6) PWA / home routing
- / auto-detects device: Windows goes to /login; others go to /m/login.
- /choose and /home_choice show manual login choice page.
- PWA manifest starts from /m/login.

Render Environment Required:
BASE_URL=https://kimssaveera.onrender.com
APP_ENV=production
